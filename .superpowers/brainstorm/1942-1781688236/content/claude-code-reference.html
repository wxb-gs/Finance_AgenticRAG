Claude Code 规划与执行模式 → 财报分析 Agent 映射

Claude Code 的规划是"追踪工具"而非"强制约束"，计划服务于对齐而非控制

Claude Code 模式

1. 规划阶段

EnterPlanMode

探索代码库 → 写 plan 文件 → 用户审批 → 退出 plan mode

Plan 是上下文文件，不是执行约束

2. 任务创建

TaskCreate × N

每个步骤一个 task，支持 blocks/blockedBy 声明依赖

Task 有状态: pending → in_progress → completed

3. 执行循环

检查 TaskList → 找下一个可执行的 → TaskUpdate(in_progress) → 执行 → TaskUpdate(completed)

独立任务通过 dispatch_subagent 并行执行

4. 自适应

执行中发现问题 → 更新 plan → 增减 task → 继续执行

计划是活的，不是写死的

5. 代码执行

Bash 工具

写脚本 → 执行 → 读输出 → 基于结果继续推理

模型决定何时需要写代码，不是预先配置

财报分析 Agent 映射

1. 规划阶段

plan_query 工具升级

模型生成结构化 Plan 对象（含步骤 + 依赖 + 预期工具）→ 写入 trace

可评测：计划是否合理？是否覆盖了所有跳？

2. 任务创建

PlanStep → SubAgentTask

每个 step 对应一个可追踪的子任务，支持依赖声明

例如：step_3 获取行业均值" 依赖 step_2 获取宁德时代数据"

3. 执行循环

主 Agent 按依赖顺序派发：无依赖的并行跑，有依赖的等待

三种子Agent: retrieval / computation / comparison

4. 自适应

检索中发现新实体/新指标 → 追加 step → 继续执行

跟 Claude Code 一样，计划随执行动态调整

5. Python 脚本执行

execute_python 工具 (NEW)

模型写 Python → 沙箱执行（pandas/numpy 可用）→ 返回结果

替代当前 computation 子Agent 的"纯 LLM 计算"

核心差异：现在 vs 目标

维度 现在 目标
规划 plan_steps 列个清单，模型后续随意发挥 Plan 对象带依赖图，任务可追踪，可评测
子Agent执行 模型手动 dispatch，无依赖管理 按 Plan 依赖自动调度，无依赖的并行、有依赖的串行
数值计算 computation 子Agent 纯 LLM 计算（不可靠） execute_python 沙箱执行（精确、可复现）
计划可观测 无 Plan 写入 trace，可评测规划合理性
