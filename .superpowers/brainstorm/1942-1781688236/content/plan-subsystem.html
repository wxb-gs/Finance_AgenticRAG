子系统 2：Plan 规划器

Claude Code 风格——Plan 是追踪工具 + 对齐手段，不是硬约束。模型参考 Plan 执行，但保留自主决策权。

核心原则

Plan 的角色：三件事
  1. 结构化分解 — 多跳查询 → 带依赖关系的步骤图，模型不用纯靠脑记
  2. 可评测追踪 — Plan 写入 trace，评测时直接判断"规划是否合理、是否覆盖所有推理跳"
  3. 对齐辅助 — 模型每轮看到 Plan 的状态，辅助判断"下一步该做什么"

不是：代码自动按 DAG 调度执行（像 PEV） 而是：模型读 Plan → 模型决定派发 → 模型可以中途改 Plan

数据结构

types.py 新增
@dataclass
class PlanStep:
    """计划中的单个步骤"""
    id: str                          # "step_1"
    description: str                 # "获取宁德时代 2024Q3 利润表"
    depends_on: list[str] = field(default_factory=list)  # ["step_3"]
    agent_type: Literal["retrieval", "computation", "comparison"] = "retrieval"
    status: Literal["pending", "in_progress", "completed", "failed"] = "pending"
    result_summary: str = ""

@dataclass
class Plan:
    """多跳查询的结构化计划"""
    query: str                       # 原始查询
    steps: list[PlanStep]            # 步骤列表
    version: int = 1                 # 计划版本（中途修改时递增）
    created_at: float = 0.0
    updated_at: float = 0.0

    def ready_steps(self) -> list[PlanStep]:
        """返回所有依赖已满足且未开始的步骤"""
        completed = {s.id for s in self.steps if s.status == "completed"}
        return [s for s in self.steps
                if s.status == "pending"
                and set(s.depends_on).issubset(completed)]

    def all_done(self) -> bool:
        return all(s.status in ("completed", "failed") for s in self.steps)
