为什么移除独立的 computation 类型？

"只能算不能搜"是一个有问题的约束

Claude Code 不会创建一个"只允许 Bash 其他什么都不许做"的子 Agent。 Python 执行跟搜索一样，都是工具——子 Agent 应该根据任务需要自由组合工具， 而不是被强制限定在单一工具上。

具体问题：computation 子 Agent 收到"计算 DuPont ROE"任务时， 如果数据里有矛盾（比如两份文档的净利润数值不同），它没有搜索能力去核实—— 只能基于主 Agent 喂给它的数据硬算。而 analysis 子 Agent 发现自己算出来的 ROE 跟某份文档里声称的 ROE 不同，可以立刻回搜核实。

类比 Claude Code：你派一个子 Agent "调查 Bug X 并修复"，它需要同时读取代码（search）和运行测试（bash）。 你不会把一个任务拆成"Agent A: 读代码"和"Agent B: 运行命令"再手动拼接结果。

覆盖度检查

财报分析任务类型 推荐子 Agent 说明
"搜索宁德时代 2024 营收" retrieval 纯搜索，无计算
"计算三家公司的 DuPont ROE 并排名" analysis 搜数据 + Python 计算 + 对比，一个 Agent 内完成
"对比宁德时代和比亚迪的研发投入趋势" analysis 横向对比 + 趋势判断，需要大模型
"找出锂电行业 ROE 最高的公司" general 先搜行业公司列表 → 逐个搜财务 → 算 ROE → 排序，多步长链路
"评估恒大退市风险" Plan → retrieval×2 + analysis 主 Agent 拆分 Plan，并行搜资产负债+违约记录，最后 analysis 综合评分
"验证报告中的 ROE 计算是否正确" analysis 搜原文 ROE 数据 → Python 重算 → 对比差异
